## 最可能被重点追问的五个方向

### 1. 资金准确性

高频追问：

- 金额精度如何保证？
- 幂等如何设计？
- 对账如何做？
- 异常如何补偿？
- 第三方支付超时如何处理？

准备重点：

```text
BigDecimal、状态机、唯一索引、幂等表、事务消息、本地消息表、实时校验、离线对账、人工补偿。
```

### 2. 大表拆分

高频追问：

- 为什么拆？
- 按什么拆？
- 怎么迁移？
- 怎么双写？
- 怎么校验？
- 怎么回滚？

准备重点：

```text
分片键、历史迁移、增量同步、双写一致性、数据校验、灰度切流、回滚预案。
```

### 3. 系统重构

高频追问：

- 旧模型具体差在哪里？
- 新模型抽象是什么？
- 如何兼容历史数据？
- 如何灰度？
- 如何证明重构收益？

准备重点：

```text
领域模型、扩展点、配置化、灰度、监控、压测、收益量化。
```

### 4. 消息一致性

高频追问：

- 消息重复怎么办？
- 消息丢失怎么办？
- 消息堆积怎么办？
- 顺序消息怎么保证？
- 消费失败怎么处理？

准备重点：

```text
至少一次投递、消费幂等、重试、死信、消费记录表、事务消息、顺序 Key、队列并行度。
```

### 5. 线上稳定性

高频追问：

- CPU 高怎么查？
- OOM 怎么查？
- Full GC 怎么查？
- 慢 SQL 怎么查？
- 线程池打满怎么查？

准备重点：

```text
top、top -Hp、jstack、jmap、MAT、jstat、GC 日志、Explain、慢查询日志、线程池监控、APM。
```
