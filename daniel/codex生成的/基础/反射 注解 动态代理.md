# 反射 注解 动态代理

## 一、先说结论

反射、注解、动态代理是 Java 框架的三件套：

```text
注解提供元数据
反射读取和操作类结构
动态代理在运行时增强行为
```

Spring、MyBatis、RPC、ORM、测试框架都大量依赖它们。

## 二、反射是什么

反射允许程序在运行时获取类的信息，并动态操作：

- Class。
- Field。
- Method。
- Constructor。
- Annotation。

常见操作：

```java
Class<?> clazz = Class.forName("com.example.User");
Object obj = clazz.getDeclaredConstructor().newInstance();
Method method = clazz.getDeclaredMethod("getName");
Object result = method.invoke(obj);
```

## 三、反射的优缺点

优点：

- 灵活。
- 解耦。
- 支持框架通用能力。
- 支持运行时动态调用。

缺点：

- 性能低于直接调用。
- 编译期不安全。
- 破坏封装。
- 代码可读性差。
- 受模块化、安全策略影响。

生产建议：

```text
框架层可以用反射，业务代码不要滥用反射。
```

## 四、注解是什么

注解是附加在类、方法、字段、参数等元素上的元数据。

注解本身不直接改变程序行为，必须由框架或工具解析。

示例：

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface AuditLog {
    String value();
}
```

## 五、注解保留策略

```text
SOURCE：只在源码中存在，编译后丢弃。
CLASS：保留到 class 文件，运行时默认不可反射读取。
RUNTIME：运行时可通过反射读取。
```

常见：

- `@Override`：SOURCE。
- 一些字节码工具注解：CLASS。
- Spring 注解：RUNTIME。

## 六、注解如何生效

注解需要处理器。

常见方式：

- 编译期注解处理器：APT。
- 运行时反射扫描。
- 字节码增强。
- Spring BeanPostProcessor。
- AOP 拦截器。

例如 `@Autowired`：

```text
Spring 扫描 Bean
-> AutowiredAnnotationBeanPostProcessor 解析注解
-> 反射注入依赖
```

## 七、JDK 动态代理

JDK 动态代理基于接口。

核心类：

- `Proxy`
- `InvocationHandler`

示例：

```java
UserService proxy = (UserService) Proxy.newProxyInstance(
    classLoader,
    new Class[]{UserService.class},
    (obj, method, args) -> {
        System.out.println("before");
        Object result = method.invoke(target, args);
        System.out.println("after");
        return result;
    }
);
```

限制：

```text
目标类必须实现接口，代理对象只能表现为接口类型。
```

## 八、CGLIB 动态代理

CGLIB 基于继承生成子类代理。

优点：

- 不要求目标类实现接口。

限制：

- final 类不能代理。
- final 方法不能增强。
- private 方法不能被正常增强。

Spring AOP 在没有接口或强制 class 代理时使用 CGLIB。

## 九、ByteBuddy

ByteBuddy 是现代字节码生成库，很多框架使用它生成或增强类。

典型使用者：

- Mockito。
- Byte Buddy Agent。
- 一些监控探针。

相比直接使用 ASM，ByteBuddy API 更友好。

## 十、Spring 和 MyBatis 如何使用代理

### Spring AOP

```text
Bean 创建后
-> BeanPostProcessor 判断是否需要增强
-> 创建代理对象
-> 调用代理对象时执行拦截器链
```

### MyBatis Mapper

```text
Mapper 接口
-> JDK 动态代理
-> MapperProxy
-> SqlSession
-> Executor
```

Mapper 接口没有实现类，就是因为代理对象在运行时生成。

## 十一、常见面试追问

### 1. 注解本身会让事务生效吗

不会。`@Transactional` 只是元数据，真正生效依赖 Spring 解析注解并创建事务代理。

### 2. 反射为什么慢

反射调用需要做方法查找、访问检查、参数封装、动态分派等，JIT 优化也不如直接调用充分。不过现代 JVM 对反射已有优化，是否成为瓶颈要看调用频率。

### 3. JDK 动态代理和 CGLIB 怎么选

有接口时可用 JDK 动态代理；没有接口时用 CGLIB。Spring Boot 里实际策略还受配置和版本影响，但本质差异是接口代理和子类代理。

## 十二、生产经验

- 注解设计要明确保留策略和作用目标。
- 运行时扫描注解不要范围过大，否则启动慢。
- 代理对象不是原始对象，排查类型转换时要注意。
- 业务代码不要依赖具体代理实现细节。
- AOP 失效常见原因是没有经过代理。
