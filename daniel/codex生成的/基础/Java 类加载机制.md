# Java 类加载机制

## 一、先说结论

Java 类加载是 JVM 把 `.class` 字节码加载到内存，并转换成可运行时使用的 `Class` 对象的过程。

完整阶段：

```text
加载 -> 验证 -> 准备 -> 解析 -> 初始化 -> 使用 -> 卸载
```

其中前五个阶段最常被问：

- 加载：把 class 文件读进 JVM。
- 验证：确保字节码安全合法。
- 准备：给静态变量分配内存并设置默认值。
- 解析：把符号引用替换为直接引用。
- 初始化：执行类构造器 `<clinit>()`，给静态变量赋真实值，执行 static 代码块。

## 二、加载阶段

加载阶段做三件事：

```text
1. 通过类全限定名获取二进制字节流。
2. 把字节流转换成方法区中的运行时数据结构。
3. 在堆中生成 java.lang.Class 对象，作为访问入口。
```

class 字节流来源可以是：

- 本地 class 文件。
- jar 包。
- 网络。
- 动态代理生成。
- CGLIB/ASM/ByteBuddy 生成。
- 加密 class 解密后加载。

## 三、验证阶段

验证是为了保证字节码不会破坏 JVM 安全。

主要包括：

- 文件格式验证。
- 元数据验证。
- 字节码验证。
- 符号引用验证。

面试表达：

```text
验证阶段保证 class 文件符合 JVM 规范，避免恶意或错误字节码破坏虚拟机。
```

## 四、准备阶段

准备阶段给类变量分配内存并设置默认值。

注意是默认值，不是代码里的真实赋值。

示例：

```java
public static int age = 18;
```

准备阶段：

```text
age = 0
```

初始化阶段：

```text
age = 18
```

特殊情况：

```java
public static final int AGE = 18;
```

如果是编译期常量，准备阶段可能直接赋值为 18。

## 五、解析阶段

解析阶段把常量池中的符号引用替换成直接引用。

符号引用：

```text
用一组符号描述目标，比如类名、方法名、字段名。
```

直接引用：

```text
能直接定位到目标的指针、偏移量或句柄。
```

## 六、初始化阶段

初始化阶段执行类构造器：

```text
<clinit>()
```

它由编译器自动收集：

- 静态变量赋值语句。
- static 代码块。

执行顺序由源码顺序决定。

示例：

```java
static int a = 1;

static {
    a = 2;
}
```

最终 `a = 2`。

## 七、什么时候触发类初始化

主动使用会触发初始化：

- `new` 对象。
- 读取或设置类的静态变量。
- 调用类的静态方法。
- 反射调用类。
- 初始化子类时，父类先初始化。
- JVM 启动时初始化主类。

不会触发初始化的典型情况：

- 通过子类引用父类静态字段，只初始化父类。
- 创建对象数组。
- 引用编译期常量。

## 八、类加载器层次

常见类加载器：

```text
Bootstrap ClassLoader：加载 JDK 核心类库。
Platform ClassLoader：加载平台类库。
Application ClassLoader：加载应用 classpath。
Custom ClassLoader：自定义类加载器。
```

JDK 8 中常说：

```text
Bootstrap -> Extension -> Application
```

JDK 9 之后模块化体系中 Extension ClassLoader 被 Platform ClassLoader 替代。

## 九、类卸载条件

类卸载必须同时满足：

```text
1. 该类所有实例都被回收。
2. 加载该类的 ClassLoader 被回收。
3. 该类对应的 Class 对象没有被引用。
```

所以普通业务类通常很难卸载，因为 Application ClassLoader 生命周期和应用一样长。

常见类卸载场景：

- 热部署。
- 插件化。
- 脚本引擎。
- 动态代理类加载器。

## 十、生产经验

- Metaspace OOM 常和类加载器泄漏、动态代理类过多有关。
- 热部署系统要重点检查旧 ClassLoader 是否被线程上下文、静态变量、ThreadLocal 持有。
- 自定义类加载器要谨慎，容易打破类隔离和 SPI 预期。
- 类是否相同不仅看全限定名，还要看是否由同一个类加载器加载。

## 十一、常见面试追问

### 1. 准备阶段和初始化阶段有什么区别

准备阶段给静态变量分配内存并赋默认值；初始化阶段执行静态赋值和 static 代码块，赋代码中的真实值。

### 2. 为什么两个类名一样的类可能不相等

JVM 判断类唯一性看“类全限定名 + 类加载器”。即使类名一样，只要类加载器不同，也被认为是不同类。

### 3. 类什么时候会卸载

类实例、Class 对象、加载它的 ClassLoader 都不可达时才可能卸载。
