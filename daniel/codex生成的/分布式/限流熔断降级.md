# 限流熔断降级

## 一、先说结论

限流、熔断、降级都是保护系统的手段，但解决的问题不同：

- 限流：限制入口流量，防止系统被打爆。
- 熔断：下游持续失败时，暂时切断调用，防止故障扩散。
- 降级：在资源不足或依赖异常时，返回兜底结果，保证核心链路可用。

一句话：

```text
限流防过载，熔断防扩散，降级保核心。
```

## 二、限流算法

### 1. 固定窗口

把时间切成固定窗口，例如每秒最多 100 次。

优点：

- 实现简单。

缺点：

- 存在临界突刺。

例如：

```text
第 1 秒末来 100 个请求，第 2 秒初又来 100 个请求。
短时间内实际承受 200 个请求。
```

### 2. 滑动窗口

把大窗口拆成多个小窗口，随着时间滑动统计。

优点：

- 比固定窗口平滑。

缺点：

- 实现复杂度更高。
- 小窗口越多，统计越精确，内存开销越大。

### 3. 漏桶

请求先进桶，桶以固定速率流出。

特点：

- 输出平滑。
- 能削峰。
- 桶满后拒绝新请求。

适合：

- 保护下游稳定处理能力。

### 4. 令牌桶

系统按固定速率生成令牌，请求拿到令牌才能通过。

特点：

- 平均速率可控。
- 允许一定突发流量。

适合：

- 大多数业务接口限流。

## 三、单机限流和分布式限流

### 单机限流

每个应用实例独立计数。

优点：

- 性能高。
- 实现简单。

缺点：

- 总限流值会随实例数变化。

### 分布式限流

多个实例共享计数。

常见实现：

- Redis + Lua。
- 网关统一限流。
- Sentinel 集群流控。

优点：

- 全局流量可控。

缺点：

- 多一次远程调用。
- Redis 或限流中心成为关键依赖。

## 四、熔断机制

熔断器通常有三种状态：

```text
CLOSED：正常调用。
OPEN：熔断打开，直接失败或走降级。
HALF_OPEN：半开，放少量请求探测恢复情况。
```

触发条件：

- 错误率过高。
- 慢调用比例过高。
- 异常数超过阈值。
- RT 超过阈值。

恢复过程：

```text
下游持续失败
-> 熔断 OPEN
-> 一段时间后 HALF_OPEN
-> 少量请求成功则恢复 CLOSED
-> 失败则重新 OPEN
```

## 五、降级策略

常见降级方式：

- 返回默认值。
- 返回缓存旧数据。
- 关闭非核心功能。
- 异步排队稍后处理。
- 提示用户稍后重试。
- 只保留核心链路。

示例：

```text
商品详情页中，库存和价格是核心，推荐商品、评论摘要可以降级。
```

## 六、线程池隔离和信号量隔离

### 1. 线程池隔离

为不同下游或业务分配独立线程池。

优点：

- 隔离彻底。
- 某个下游卡死不会耗尽主线程池。

缺点：

- 线程开销大。
- 参数配置复杂。

### 2. 信号量隔离

用信号量限制并发数。

优点：

- 开销小。
- 性能好。

缺点：

- 无法隔离慢调用造成的线程阻塞。

## 七、Sentinel 核心概念

Sentinel 常见能力：

- 流量控制。
- 熔断降级。
- 系统自适应保护。
- 热点参数限流。
- 集群流控。

核心概念：

- Resource：被保护资源。
- Rule：规则。
- Entry：资源访问入口。
- Slot Chain：责任链处理统计、限流、降级等逻辑。

常见规则：

- QPS 限流。
- 并发线程数限流。
- 慢调用比例熔断。
- 异常比例熔断。
- 异常数熔断。

## 八、限流位置选择

可以放在：

- CDN。
- 网关。
- 应用入口。
- 具体方法。
- 下游调用客户端。
- MQ 消费端。

原则：

```text
越靠前越能节省资源，越靠后越贴近业务语义。
```

生产中通常组合使用：

```text
网关粗粒度限流 + 应用内细粒度限流 + 下游调用熔断
```

## 九、常见面试追问

### 1. 熔断和降级有什么区别

熔断是判断依赖不可用后自动切断调用；降级是切断后或资源不足时返回兜底逻辑。熔断是一种触发机制，降级是一种处理结果。

### 2. 限流后请求怎么处理

可以拒绝、排队、降级、异步化。高并发核心接口通常快速失败或返回友好提示，避免排队拖垮系统。

### 3. 为什么线程池隔离能防雪崩

因为每个依赖有独立资源池，某个下游慢或挂，不会占满整个应用的工作线程。

## 十、生产经验

- 限流阈值必须来自压测和线上指标，不要拍脑袋。
- 限流要有明确的用户体验设计。
- 熔断阈值要避免过于敏感导致频繁抖动。
- 降级数据要能被业务接受，不能返回误导性结果。
- 限流、熔断、降级都必须有监控和告警。
