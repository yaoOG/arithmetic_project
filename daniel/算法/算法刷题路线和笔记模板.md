# 算法刷题路线和笔记模板

## 一、先说结论

Java 资深开发不一定需要刷大量难题，但高频中等题型必须稳定：

```text
数组/双指针
链表
栈和队列
哈希表
二分查找
滑动窗口
回溯
动态规划
堆
图 BFS/DFS/拓扑排序
```

目标不是背答案，而是形成题型识别能力：

```text
看到题目 -> 判断模型 -> 套用模板 -> 处理边界 -> 写出可运行 Java 代码。
```

## 二、推荐刷题顺序

### 第一阶段：基础数据结构

- 数组。
- 链表。
- 哈希表。
- 栈。
- 队列。

目标：

```text
熟悉 Java 集合、指针移动、边界处理。
```

### 第二阶段：核心技巧

- 双指针。
- 滑动窗口。
- 前缀和。
- 二分查找。
- 单调栈。
- 堆。

目标：

```text
掌握高频优化手段，把 O(n^2) 降到 O(n) 或 O(nlogn)。
```

### 第三阶段：递归和搜索

- 二叉树 DFS/BFS。
- 回溯。
- 图 DFS/BFS。
- 拓扑排序。

目标：

```text
能写清递归终止条件、状态选择、回退逻辑。
```

### 第四阶段：动态规划

- 一维 DP。
- 二维 DP。
- 背包。
- 子序列。
- 区间 DP。

目标：

```text
能定义状态、写转移方程、确定初始化和遍历顺序。
```

## 三、每道题笔记模板

建议每道题都按下面结构写：

```text
# 题目名

## 一、题目类型

数组 / 链表 / 二分 / DP / 回溯 / 图

## 二、核心思路

用 3-5 句话说明为什么这样做。

## 三、边界条件

- 空数组。
- 单元素。
- 重复元素。
- 越界。
- 溢出。

## 四、Java 代码

代码必须能独立通过。

## 五、复杂度

时间复杂度：
空间复杂度：

## 六、相似题

列出同类题。
```

## 四、题型判断信号

### 1. 双指针

题目信号：

- 有序数组。
- 两数之和变种。
- 删除重复元素。
- 快慢指针。
- 原地移动元素。

常见模板：

```java
int left = 0, right = nums.length - 1;
while (left < right) {
    if (condition) {
        left++;
    } else {
        right--;
    }
}
```

### 2. 滑动窗口

题目信号：

- 连续子数组。
- 最长/最短子串。
- 满足某种条件的窗口。

模板：

```java
int left = 0;
for (int right = 0; right < n; right++) {
    // 加入 right
    while (窗口不满足条件) {
        // 移出 left
        left++;
    }
    // 更新答案
}
```

### 3. 二分查找

题目信号：

- 有序。
- 最大值最小化。
- 最小值最大化。
- 查找边界。

模板：

```java
int left = 0, right = n - 1;
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] == target) {
        return mid;
    } else if (nums[mid] < target) {
        left = mid + 1;
    } else {
        right = mid - 1;
    }
}
return -1;
```

### 4. 回溯

题目信号：

- 所有组合。
- 所有排列。
- 子集。
- 棋盘搜索。
- 需要撤销选择。

模板：

```java
void backtrack(路径, 选择列表) {
    if (满足结束条件) {
        result.add(路径);
        return;
    }

    for (选择 : 选择列表) {
        做选择;
        backtrack(路径, 选择列表);
        撤销选择;
    }
}
```

### 5. 动态规划

题目信号：

- 最值。
- 方案数。
- 是否可达。
- 子序列。
- 不能贪心直接确定。

四步：

```text
1. 定义 dp[i] 的含义。
2. 写状态转移方程。
3. 初始化。
4. 确定遍历顺序。
```

## 五、Java 常用结构

### 栈

```java
Deque<Integer> stack = new ArrayDeque<>();
stack.push(x);
stack.pop();
stack.peek();
```

### 队列

```java
Queue<Integer> queue = new ArrayDeque<>();
queue.offer(x);
queue.poll();
queue.peek();
```

### 优先队列

```java
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
```

注意：

```text
(a, b) -> b - a 可能整数溢出，更稳妥写法是 Integer.compare(b, a)。
```

### 哈希表

```java
Map<Integer, Integer> map = new HashMap<>();
map.put(key, value);
map.getOrDefault(key, 0);
map.containsKey(key);
```

## 六、面试刷题标准

一题真正掌握的标准：

- 能说出题型。
- 能说出为什么这么做。
- 能手写代码。
- 能解释复杂度。
- 能处理边界。
- 能举出 1-2 道相似题。

不要只看题解。至少隔天复写一次。
