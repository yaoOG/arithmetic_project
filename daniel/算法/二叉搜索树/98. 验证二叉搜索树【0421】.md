判断一个二叉树是否为有效的二叉搜索树（BST），最容易踩的坑是**只检查当前节点与其直接子节点的关系**（即只检查 `root.left.val < root.val < root.right.val`）。

实际上，BST 的性质具有**传递性**：左子树的**所有**节点都必须小于根节点，右子树的**所有**节点都必须大于根节点。

### 中序遍历（最符合直觉）

这是利用了 BST 的一个核心数学性质：**有效 BST 的中序遍历序列一定是严格递增的。**

我们只需要按照“左 -> 根 -> 右”的顺序遍历整棵树，并记录前一个遍历到的节点值。如果当前节点值小于或等于前一个节点值，则不是 BST。

#### Java 实现（迭代法）

这种方式在实际面试中往往更受青睐，因为它能展示你对栈（Stack）和树遍历的熟练掌握。



```Java
class Solution {
    public boolean isValidBST(TreeNode root) {
        Deque<TreeNode> stack = new ArrayDeque<>();
        Integer prev = null; // 记录前一个节点的值

        while (!stack.isEmpty() || root != null) {
            // 1. 尽可能向左走
            while (root != null) {
                stack.push(root);
                root = root.left;
            }
            
            // 2. 弹出并处理当前节点
            root = stack.pop();
            
            // 如果当前节点值小于等于前一个值，不合法
            if (prev != null && root.val <= prev) {
                return false;
            }
            prev = root.val;

            // 3. 转向右子树
            root = root.right;
        }
        return true;
    }
}
```

---

### 复杂度分析

- **时间复杂度**：$O(n)$，其中 $n$ 是节点个数。每个节点都需要被访问一次。
    
- **空间复杂度**：$O(n)$。在最坏情况下（树呈链状），递归栈或显式栈的深度为 $n$。
