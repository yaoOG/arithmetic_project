题目：给你二叉搜索树的根节点 `root` ，该树中的 **恰好** 两个节点的值被错误地交换。_请在不改变其结构的情况下，恢复这棵树_ 。

这道题是 BST 性质的进阶应用。既然是“恢复”一棵 BST，最核心的切入点依然是那条黄金法则：**有效 BST 的中序遍历序列必须是严格递增的。**

如果其中有两个节点被交换了，那么原本递增的序列中就会出现**逆序对**（即当前值比前一个值小）。

---

### 1. 核心逻辑：寻找逆序对

假设正常的 BST 中序遍历是 `[1, 2, 3, 4, 5, 6]`。

如果有两个节点被错误交换，会出现两种情况：

- **情况 A：两个节点相邻**
    
    - 例如交换 2 和 3：`[1, 3, 2, 4, 5, 6]`
        
    - **现象**：产生 **1 个** 逆序对 `(3, 2)`。
        
- **情况 B：两个节点不相邻**
    
    - 例如交换 2 和 5：`[1, 5, 3, 4, 2, 6]`
        
    - **现象**：产生 **2 个** 逆序对 `(5, 3)` 和 `(4, 2)`。
        
    - **规律**：我们要找的第一个错误节点是第一个逆序对的**较大值**（5），第二个错误节点是最后一个逆序对的**较小值**（2）。
        

---

### 2. 算法实现 (Java)

我们使用隐式中序遍历（递归）来解决。在遍历过程中，维护一个 `prev` 指针记录上一个访问的节点，并用 `first` 和 `second` 记录那两个“犯错”的节点。

```Java
class Solution {
    private TreeNode first = null;
    private TreeNode second = null;
    private TreeNode prev = null;

    public void recoverTree(TreeNode root) {
        // 1. 中序遍历寻找错误节点
        traverse(root);

        // 2. 交换两个错误节点的值
        if (first != null && second != null) {
            int temp = first.val;
            first.val = second.val;
            second.val = temp;
        }
    }

    private void traverse(TreeNode root) {
        if (root == null) return;

        traverse(root.left);

        // --- 中序处理逻辑 ---
        if (prev != null && root.val < prev.val) {
            // 如果是第一次发现逆序，第一个节点选较大的那个(prev)
            if (first == null) {
                first = prev;
            }
            // 无论第一次还是第二次发现逆序，第二个节点都选较小的那个(root)
            // 这样如果是相邻交换，second 也会被正确赋值
            second = root;
        }
        prev = root;
        // -----------------

        traverse(root.right);
    }
}
```

### 3. 架构师视角的进阶思考

#### 空间复杂度与 Morris 遍历

上面的递归方法空间复杂度是 $O(H)$（$H$ 为树高，最坏 $O(n)$）。面试官可能会问：**“能否用 $O(1)$ 的空间实现？”**

要达到常数级空间，需要使用 **Morris 中序遍历**。它的核心原理是利用树中的空闲叶子节点指针（Right Pointers）指向中序遍历的后继节点，从而实现不依赖栈的遍历。
